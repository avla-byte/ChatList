# Схема базы данных ChatList

Файл БД: SQLite, один файл (например `chatlist.sqlite` или путь из настроек).  
Кодировка текста: UTF-8. Рекомендуется при открытии соединения включать `PRAGMA foreign_keys = ON`.

API-ключи **не** хранятся в таблицах: в `models` задаётся только **имя переменной окружения**, из которой ключ читается при запросе (см. `.env`).

---

## Обзор таблиц

| Таблица   | Назначение |
|-----------|------------|
| `prompts` | Сохранённые в библиотеке запросы пользователя |
| `models`  | Нейросети: URL, ссылка на env-ключ, активность |
| `results` | Постоянно сохранённые пары «промт — модель — ответ» (по выбранным строкам) |
| `settings` | Ключ–значение: параметры программы (пути, UI и т.д.) |

---

## Таблица `prompts`

Запросы, которые пользователь **сохранил** для повторного выбора.

| Поле         | Тип        | Ограничения        | Описание |
|--------------|------------|--------------------|----------|
| `id`         | INTEGER    | PRIMARY KEY, AUTOINCREMENT | Суррогатный ключ |
| `created_at` | TEXT / INTEGER | NOT NULL, DEFAULT  | Момент записи (ISO-8601 или Unix time) |
| `body`       | TEXT       | NOT NULL          | Текст промта |
| `tags`       | TEXT       | NULL              | Теги: строка, разделитель (например запятая) или JSON-массив — на усмотрение реализации |

**Индексы (опционально, для поиска/сортировки):** `created_at`, при необходимости полнотекстовый по `body` — в SQLite через `FTS5` отдельной таблицей, если понадобится.

---

## Таблица `models`

Список нейросетей, в которые уходят запросы.

| Поле        | Тип     | Ограничения | Описание |
|-------------|---------|-------------|----------|
| `id`        | INTEGER | PRIMARY KEY, AUTOINCREMENT | Суррогатный ключ |
| `name`      | TEXT    | NOT NULL    | Отображаемое имя (колонка в таблице ответов) |
| `api_url`   | TEXT    | NOT NULL    | Полный URL endpoint (в реализации — обычно `…/v1/chat/completions` в стиле OpenAI) |
| `api_id`    | TEXT    | NOT NULL    | **Имя переменной окружения** для ключа, не сам ключ (например `OPENAI_API_KEY`) |
| `api_model` | TEXT    | NOT NULL, по умолчанию `''` | Идентификатор модели у провайдера (параметр `model` в теле запроса) |
| `is_active` | INTEGER | NOT NULL, DEFAULT 1 | 1 — участвует в рассылке, 0 — отключена |

**Индексы:** `is_active` (для `WHERE is_active = 1`).

**Миграция:** схема версии 2 добавляет `api_model` к существующим БД (см. `db.py`).

**Заметка:** при расширении (OpenAI, Groq, DeepSeek) можно добавить поле `provider` / `config_json` без изменения базовой смысловой схемы.

---

## Таблица `results`

Сохранённые **избранные** ответы после нажатия «Сохранить» в UI.  
В спецификации явно перечислены не все поля; ниже — логичный минимум, чтобы однозначно восстановить контекст и связать с моделью (и при желании с записью в `prompts`).

| Поле            | Тип     | Ограничения | Описание |
|-----------------|---------|-------------|----------|
| `id`            | INTEGER | PRIMARY KEY, AUTOINCREMENT | Суррогатный ключ |
| `created_at`    | TEXT / INTEGER | NOT NULL, DEFAULT | Когда запись сделана |
| `model_id`      | INTEGER | NOT NULL, FK → `models(id)` | Какая нейросеть дала ответ |
| `prompt_id`     | INTEGER | NULL, FK → `prompts(id)`  | Ссылка на библиотечный промт, если применимо |
| `prompt_snapshot` | TEXT  | NOT NULL   | **Копия текста запроса** в момент сохранения (нужна, если промт вводили вручную и не выносили в `prompts`, и для стабильной истории) |
| `response_text` | TEXT    | NOT NULL   | Сохранённый текст ответа |

**Поведение при удалении:**

- Вариант A: `ON DELETE` для `model_id` / `prompt_id` — `RESTRICT` (нельзя удалить модель/промт, пока есть `results`) или `SET NULL` для `prompt_id` только, если договорились снимать ссылку.
- Вариант B: «мягкое» удаление моделей — тогда `results` не ломаются, если `model_id` остаётся, а имя подставляется из архива. Для первой версии чаще достаточно **RESTRICT** на `model_id` или запрет физического удаления используемых моделей.

**Индексы:** `created_at`, `model_id`, при необходимости `prompt_id`.

---

## Таблица `settings`

Произвольные настройки приложения (путь к файлу БД, размеры окна, поведение по умолчанию — что решит `main.py` / конфиг).

| Поле   | Тип  | Ограничения | Описание |
|--------|------|-------------|----------|
| `key`  | TEXT | PRIMARY KEY | Уникальное имя настройки |
| `value`| TEXT | NOT NULL    | Значение (строка; JSON внутри строки — при необходимости) |

---

## Связи (ER, логически)

```
prompts 1───* results (optional FK prompt_id)
models  1───* results (required FK model_id)
```

Временная таблица ответов **в RAM** в SQLite не хранится и в эту схему не входит.

---

## Согласованность с `PROJECT.md`

- **prompts:** дата, промт, теги — отражено в `created_at`, `body`, `tags`.
- **models:** `name`, URL, ссылка на env (`api_id`), `is_active` — отражено; ключи только в `.env`.
- **results** и **settings** в спецификации перечислены без детализации полей; предложенная схема обеспечивает сохранение отмеченных строк вместе с идентификатором модели и привязкой к смыслу промта.

При реализации в `db.py` схему можно реализовать через `CREATE TABLE` + миграции (версия схемы в `settings`, например ключ `schema_version`).
