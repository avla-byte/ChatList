# План реализации ChatList

Поэтапный порядок работ на основе [PROJECT.md](./PROJECT.md).

---

## Этап 0. Подготовка проекта

1. Зафиксировать структуру каталогов: модули `db.py`, `models.py`, `network.py`, `main.py`, при необходимости пакет `src/` или плоская схема — единообразно на весь проект.
2. Создать `requirements.txt` (PyQt6 или PyQt5, httpx или requests, python-dotenv для `.env`).
3. Добавить `.env.example` с именами переменных для ключей API (без реальных значений) и описанием, что ключи не хранятся в БД, а только ссылки на переменные в таблице `models`.
4. Добавить `.gitignore` (`.env`, `__pycache__`, БД при необходимости локального игнорирования).

---

## Этап 1. Схема БД и модуль `db.py`

1. Спроектировать таблицы:
   - **prompts** — дата, текст промта, теги;
   - **models** — name, api-url, api-id (идентификатор/имя переменной ключа в `.env`), is_active;
   - **results** — поля для связи с промтом, моделью и сохранённым ответом (уточнить при реализации: внешние ключи, текст ответа, метаданные);
   - **settings** — ключ–значение или иная схема под настройки UI/поведения.
2. Реализовать инициализацию SQLite (создание файла, `CREATE TABLE IF NOT EXISTS`).
3. Реализовать функции/класс репозитория: CRUD для `prompts` и `models`, запись в `results`, чтение/запись `settings`.
4. Все обращения к SQLite — **только** в `db.py`; логирование ошибок БД.

---

## Этап 2. Модуль `network.py`

1. Общий слой HTTP: таймауты, базовая обработка сетевых ошибок, логирование запросов/ошибок (без утечки ключей в лог).
2. Функция(и) вида «вызвать эндпоинт модели с телом запроса и заголовками», куда подставляется ключ из окружения по имени из `models.api-id`.
3. Валидация входных параметров (URL, пустой промт и т.д.) — на границе модуля или вызывающего кода.

---

## Этап 3. Модуль `models.py`

1. Загрузка списка активных моделей из БД (`is_active = 1`).
2. Для каждой модели: разрешение ключа из `os.environ` по полю api-id; при отсутствии ключа — предсказуемая ошибка и запись в лог.
3. Вызов `network.py` для параллельной или последовательной отправки промта во все активные модели; сбор результатов в единую структуру для UI (модель, текст ответа, флаг ошибки/таймаута).
4. Не смешивать с PyQt: только данные и вызовы сети/БД.

---

## Этап 4. Временные результаты (в памяти)

1. Определить структуру «сессии ответов»: список строк (название модели, текст, `selected: bool`), **не** в SQLite.
2. Правила: при новом промте — полная очистка и перезаполнение после получения ответов; при «Сохранить» — перенос отмеченных строк в `results` через `db.py`, затем очистка временной таблицы.

---

## Этап 5. GUI в `main.py` (PyQt)

1. Окно ввода: многострочное поле промта, выбор сохранённого промта из `prompts` (загрузка списка из БД).
2. Кнопка «Отправить» / аналог: вызов логики из `models.py`, отображение прогресса (индикатор или статус).
3. Таблица результатов: колонки — модель, ответ, чекбокс `selected`.
4. Кнопка «Сохранить»: для всех строк с `selected = True` — запись в `results`, затем очистка временной таблицы в памяти и обновление UI.
5. Обработка ошибок сети/БД: понятные сообщения пользователю (QMessageBox или строка статуса).

---

## Этап 6. Интеграция и проверка

1. Сценарий: один активный мок/тестовая модель с фиктивным URL или локальным стендом — проверка цепочки БД → сеть → таблица → сохранение.
2. Сценарий: несколько моделей, часть с неверным ключом — корректное отображение ошибки по строке, не падение всего окна.
3. Проверка «новый запрос»: старая временная таблица не смешивается с новой.

---

## Этап 7. Опциональные доработки (после базового релиза)

1. Поиск и сортировка по таблицам в UI (и при необходимости запросы в `db.py`).
2. Экспорт выбранных результатов в Markdown / JSON.
3. Адаптеры под разные провайдеры (OpenAI, DeepSeek, Groq) в `network.py` / `models.py` — единый контракт, разные тела запросов.
4. Расширенные логи запросов (файл ротации, уровни логирования).

---

## Порядок зависимостей (кратко)

`db.py` и `.env` → `network.py` → `models.py` → временная модель в памяти + `main.py` → тесты сценариев → опции из раздела 6 спецификации.
